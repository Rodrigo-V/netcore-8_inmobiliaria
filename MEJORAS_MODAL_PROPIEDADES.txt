============================================
   CAMBIOS IMPLEMENTADOS
============================================

✅ 1. TIPO DE PROPIEDAD → SELECT
   - Convertido de campo de texto a SELECT
   - Datos obtenidos desde SP: PP_psnp_Propiedad_ObtenerTipos
   - Opciones: Casa, Departamento, Oficina, Local Comercial, Terreno, Parcela, Bodega, Estacionamiento, Elemento

✅ 2. REGIÓN → SELECT (CON API PÚBLICA)
   - Convertido a SELECT
   - Usa API pública de Chile: https://apis.digital.gob.cl/dpa/
   - 16 regiones de Chile cargadas
   - Campo obligatorio (*)

✅ 3. COMUNA → SELECT DEPENDIENTE
   - Convertido a SELECT
   - DEPENDIENTE de la selección de región
   - Se habilita solo al seleccionar región
   - Carga comunas desde API pública de Chile por región
   - Carga dinámica mediante AJAX

✅ 4. DORMITORIOS → SELECT
   - Convertido de campo de texto a SELECT
   - Opciones: 0 (Sin dormitorios), 1, 2, 3, 4, 5, 6+

✅ 5. BAÑOS → SELECT
   - Convertido de campo de texto a SELECT
   - Opciones: 0, 1, 2, 3, 4, 5+

✅ 6. ESTADO → SELECT
   - Convertido de campo de texto a SELECT
   - Datos obtenidos desde SP: PP_psnp_Propiedad_ObtenerEstados
   - Valor por defecto: "Disponible"
   - Campo obligatorio (*)
   - Opciones: Disponible, Reservada, Vendida, Arrendada, En Accion, En Carpeta, En Edicion, Suspendida, etc.

✅ 7. AGENTE RESPONSABLE → SELECT
   - Convertido de campo de texto a SELECT
   - Datos obtenidos desde SP: PP_psnp_Propiedad_ObtenerAgentes
   - Lista de usuarios activos del sistema (Nombres + Apellidos)


============================================
   STORED PROCEDURES CREADOS
============================================

✅ PP_psnp_Propiedad_ObtenerTipos
   - Retorna lista de tipos de propiedad
   - Sin parámetros
   - Columna: Tipo

✅ PP_psnp_Propiedad_ObtenerEstados
   - Retorna lista de estados de propiedad
   - Sin parámetros
   - Columna: Estado
   - Combina estados existentes en BD + estados predefinidos

✅ PP_psnp_Propiedad_ObtenerAgentes
   - Retorna lista de agentes responsables
   - Obtiene usuarios activos de la tabla Usuarios
   - Sin parámetros
   - Columna: Agente (Nombres + Apellidos)

============================================
   FUNCIONALIDAD REGIÓN-COMUNA
============================================

1. Usuario selecciona REGIÓN
   ↓
2. JavaScript detecta el cambio (evento 'change')
   ↓
3. Se obtiene el código de región
   ↓
4. Se hace llamada AJAX a la API pública de Chile
   URL: https://apis.digital.gob.cl/dpa/regiones/{codigo}/comunas
   ↓
5. Se reciben las comunas en formato JSON
   ↓
6. Se llenan las opciones del SELECT de comunas
   ↓
7. Se habilita el SELECT de comunas

============================================
   EJEMPLO DE USO
============================================

1. Usuario hace clic en "Agregar Propiedad"
2. Se abre el modal con SweetAlert2
3. Se cargan automáticamente:
   - Tipos de propiedad (desde BD)
   - Estados (desde BD)
   - Agentes (desde BD)
   - Regiones (hardcoded, 16 regiones de Chile)
4. Usuario selecciona región (ej: "Metropolitana de Santiago")
5. Se cargan automáticamente las comunas desde la API
6. Usuario completa los demás campos
7. Hace clic en "Guardar"
8. Se validan los campos obligatorios
9. Se envía al servidor
10. Se guarda en la BD
11. Se actualiza la grilla
