============================================
   IMPORTANTE: FLUJO DE TRABAJO
============================================

⚠️  Los Clientes Match NO se crean manualmente

FLUJO:
1. Un cliente contacta como Lead
2. Se registra en "Clientes Leads"
3. Cuando avanza en el proceso, se convierte a "Match"
4. El sistema automáticamente crea el registro en Clientes_Match
5. Desde "Clientes Match" solo se MODIFICA o ELIMINA


============================================
   PRÓXIMOS PASOS PARA USAR
============================================

1. EJECUTAR LOS STORED PROCEDURES
   Opción A - Individual:
   - Abrir cada archivo .sql en SQL Server Management Studio
   - Ejecutarlos uno por uno

   Opción B - Automático:
   - Abrir SQL/00_Ejecutar_Todos_SPs_ClientesMatch.sql
   - Ejecutar (creará todos los SPs)

2. COMPILAR Y EJECUTAR LA APLICACIÓN
   > cd C:\Users\rvilchez\Desktop\proySQL\Inmobiliaria.Net8
   > dotnet build
   > dotnet run

3. ACCEDER AL MÓDULO
   - Iniciar sesión en la aplicación
   - Ir al menú lateral: GENERAL > Clientes Match

4. PROBAR LA FUNCIONALIDAD
   a) Ir a "Clientes Leads"
   b) Seleccionar un Lead
   c) Convertirlo a Match
   d) Ir a "Clientes Match"
   e) Verificar que aparece el cliente
   f) Modificar sus datos
   g) Ver que se actualizan correctamente


============================================
   ESTRUCTURA DE LA TABLA Clientes_Match
============================================

Campo                 Tipo            Descripción
------------------    ------------    ------------------------
ID_Interno           NVARCHAR(50)    ID único (PK)
Tipo_Match           NVARCHAR(100)   Tipo de match
Nombre               NVARCHAR(200)   Nombre completo
Rut                  NVARCHAR(20)    RUT del cliente
Datos_adjuntos       NVARCHAR(MAX)   Observaciones
Direccion            NVARCHAR(200)   Dirección
Comuna               NVARCHAR(100)   Comuna
Estado_Civil         NVARCHAR(50)    Estado civil
Profesion            NVARCHAR(100)   Profesión
Telefono             NVARCHAR(20)    Teléfono
Correo               NVARCHAR(100)   Email
Giro_Razon_Social    NVARCHAR(200)   Giro o razón social


============================================
   ENDPOINTS API DISPONIBLES
============================================

GET  /ClientesMatch
     → Vista principal

GET  /ClientesMatch/Listar
     → Lista todos los clientes match con filtros

GET  /ClientesMatch/ObtenerPorId/{id}
     → Obtiene un cliente match por ID

PUT  /ClientesMatch/Actualizar
     → Actualiza un cliente match

DELETE /ClientesMatch/Eliminar/{id}
     → Elimina un cliente match

GET  /ClientesMatch/ContarTotal
     → Cuenta total de clientes match
